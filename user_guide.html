Habit Managementïƒ

Adding Habitsïƒ

Create new habits with the add command:

Best Practices: - Use clear, specific names - Write actionable task descriptions - Start with achievable goals

Listing Habitsïƒ

View all your habits:

Output shows name, task description, and periodicity for each habit.

Deleting Habitsïƒ

Remove habits you no longer want to track:

Youâ€™ll be prompted for confirmation. This permanently removes the habit and all its completion history.

Completing Habitsïƒ

Mark habits as complete when you finish them:

Important Notes: - Daily habits can only be completed once per day - Weekly habits can only be completed once per week (Monday-Sunday) - Completions are timestamped for accurate tracking

Advanced Featuresïƒ

Data Storageïƒ

Habit data is stored locally in: - Linux/macOS: ~/.config/grit-guardian/habits.db - Windows: %APPDATA%\\grit-guardian\\habits.db

The database is a standard SQLite file that you can: - Back up by copying the file - Examine with SQLite tools - Reset by deleting the file (then run gg init)

Sample Dataïƒ

The init command creates sample habits:

This only works on empty databases. If you already have habits, it shows your current status instead.
